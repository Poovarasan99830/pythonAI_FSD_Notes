# ------------------------------------------
# 🔹 1️⃣ Git
# ------------------------------------------

### What

* **Git** is a **version control system**.
* It tracks changes in your code.

### Why

* To track code history.
* To work in teams.
* To undo mistakes.

### When

* When building any software project.
* From the start of development.

### Where

* Works on your **local machine**.

### How (Basic Commands)

```bash
git init              # Initialize repo
git add .             # Add files
git commit -m "msg"   # Save changes
git status            # Check status
git log               # See history
```



# ------------------------------------------
# 🔹 2️⃣ GitHub
# ------------------------------------------

### What

* **GitHub** is a **cloud platform** to store Git repositories online.

### Why

* Backup your code.
* Share with team.
* Deploy projects.

### When

* When you want remote storage or collaboration.

### Where

* Online platform:
  👉 GitHub

### How (Connect Git to GitHub)

```bash
git remote add origin https://github.com/username/repo.git
git push -u origin main
```


# ------------------------------------------
# 🔹 Git vs GitHub (Simple Difference)
# ------------------------------------------

| Git            | GitHub             |
| -------------- | ------------------ |
| Local tool     | Online platform    |
| Tracks changes | Stores code online |
| Works offline  | Needs internet     |



# ------------------------------------------
#  Important Concepts
# ------------------------------------------

* Repository (Repo) → Project folder
* Branch → Separate version of code
* Merge → Combine branches
* Clone → Copy repo
* Pull → Get latest changes
* Push → Upload changes
