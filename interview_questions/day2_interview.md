# # What is SOLID Principle?
# # _____________________________________________

# SOLID principles are design guidelines in OOP that help build scalable, maintainable, and flexible software systems.



# SOLID is a set of 5 object-oriented design principles that help us write:

# Clean code
# Scalable code
# Easy-to-maintain code
# Testable code
# Used a lot in Python, Java, C#, Django, Flask projects.

# S – Single Responsibility Principle (SRP)
# O – Open/Closed Principle (OCP) -->📌 Open for extension, closed for modification
# L – Liskov Substitution Principle (LSP) 📌 Child class should replace parent class without breaking
# I – Interface Segregation Principle (ISP) 📌 Don’t force classes to implement unused methods
# D – Dependency Inversion Principle (DIP)  📌 Depend on abstraction, not concrete class
